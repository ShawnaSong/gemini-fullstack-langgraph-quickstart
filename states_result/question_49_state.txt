*   SWE-bench is a benchmark used to evaluate the ability of large language models (LLMs) and AI agents to resolve real-world software issues. [dev](https://vertexaisearch.cloud.google.com/id/0-1) [github](https://vertexaisearch.cloud.google.com/id/0-3) [openai](https://vertexaisearch.cloud.google.com/id/0-0)
*   The benchmark uses real GitHub issues and pull requests from open-source Python repositories. [anthropic](https://vertexaisearch.cloud.google.com/id/0-4) [dev](https://vertexaisearch.cloud.google.com/id/0-1) The AI's solution is evaluated by running the repository's tests to see if the issue is resolved. [anthropic](https://vertexaisearch.cloud.google.com/id/0-4)
*   SWE-bench Verified is a human-validated subset of SWE-bench with 500 problems that are confirmed to be solvable. [dev](https://vertexaisearch.cloud.google.com/id/0-1) This subset aims to provide a more reliable evaluation of AI models by addressing issues in the original SWE-bench dataset, such as overly specific unit tests or poorly specified issue descriptions.
*   Key metrics for evaluating performance on SWE-bench include the resolution rate (percentage of submitted instances successfully resolved), the number of instances submitted, completed, and resolved.

Here's a breakdown of the types of issues addressed and some limitations:

**Types of Bugs and Software Issues Addressed:**

*   **Complex Reasoning:** The tasks demand complex reasoning beyond simple code generation, including interaction with execution environments. [zencoder](https://vertexaisearch.cloud.google.com/id/2-4)
*   **Test Integration:** Models must understand, modify, and test the code, with solutions graded against real unit tests from the pull request that closed the original GitHub issue.
*   **New Feature Requests:** Each issue can be either a bug report or a new feature request.

**SWE-bench Verified Focus:**

*   SWE-bench Verified is a refined subset of SWE-bench, designed to provide a more reliable evaluation of AI models. *   SWE-bench Verified primarily tests models' ability to fix simple bugs in a small set of well-known repositories. [epoch](https://vertexaisearch.cloud.google.com/id/2-6)

**Limitations and Challenges:**

*   **Weak Testing:** Some tasks in SWE-bench may have inadequate unit tests, leading to false positives where models appear to solve problems but haven't truly addressed the underlying issues. [zencoder](https://vertexaisearch.cloud.google.com/id/2-4)
*   **Data Contamination:** Issues in SWE-bench may have been created before the training cutoff dates of several LLMs, potentially leading to data leakage and inflated performance scores. [zencoder](https://vertexaisearch.cloud.google.com/id/2-4)
*   **Lack of Real-World Representation:** SWE-bench primarily focuses on Python repositories, which may not fully represent the diversity of programming languages and software engineering tasks encountered in real-world development. [zencoder](https://vertexaisearch.cloud.google.com/id/2-4) [epoch](https://vertexaisearch.cloud.google.com/id/2-6)
*   **Benchmark Specificity:** SWE-bench scores can sometimes reflect a model's strength on known datasets rather than its ability to generalize to new or different coding scenarios. [zencoder](https://vertexaisearch.cloud.google.com/id/2-4)
*   **Solution Leakage:** A significant portion of the issues in SWE-Bench have solutions directly provided in the issue report or comments. [runloop](https://vertexaisearch.cloud.google.com/id/2-7) [arxiv](https://vertexaisearch.cloud.google.com/id/2-8)
*   **Overemphasis on Bug Fixing:** SWE-Bench primarily focuses on bug fixing and doesn't adequately assess LLMs' ability to generate new code or design software architectures. [runloop](https://vertexaisearch.cloud.google.com/id/2-7)
*   **Difficulty in Setting Up Environments:** It is sometimes difficult to reliably set up the SWE-bench development environments for the agents, inadvertently causing unit tests to fail regardless of the solution. [openai](https://vertexaisearch.cloud.google.com/id/2-2)

SWE-bench and SWE-bench Verified are valuable tools for evaluating AI models on software engineering tasks, but it's essential to be aware of their limitations and potential biases.
